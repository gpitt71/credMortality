# credMortality: reproducibility README

This folder contains the code and intermediate results used to reproduce the code-generated outputs for the manuscript

**“Forecasting Sub-Population Mortality Using Credibility Theory”**

The supplement is organized as an R package together with a `targets` pipeline. The code generates synthetic mortality data from publicly available Human Mortality Database (HMD) life-table information for Italy, then fits the mortality models and produces the figures and saved result files used in the manuscript.

## 1. What this supplement reproduces

This supplement reproduces the code-generated manuscript outputs currently referenced in `main-text-revision.tex`:

- the four-panel credibility illustration figure
- the nine sample-size plots
- the three MSEP fan plots and their appendix versions
- the four rolling-window MSE panels and four rolling-window deviance panels
- the appendix versions of the rolling-window MSE and deviance panels
- the mortality-trend panel used in the appendix Lexis figure
- the main saved simulation outputs:
  - `output/results_age_breakets_smooth.csv`
  - `output/lc_msep_results.rds`

It also includes a precomputed file

- `output/convergence_separate_population_models.csv`

which is shipped with the supplement, but is **not currently regenerated by the present `_targets.R` pipeline**.

## 2. Data source and provenance

The synthetic data used throughout the supplement are generated from female mortality and exposure data for Italy obtained through the Human Mortality Database (HMD), accessed in the function `data_generator_hmd_lt()` via `demography::hmd.mx(country = "ITA", ...)`.

Accordingly:

- the supplement does **not** use confidential data
- the supplement uses **synthetic data generated from HMD-derived mortality schedules**
- reproducing the results requires valid HMD login credentials

Please see <https://www.mortality.org/> for data access conditions and registration.

## 5. Required manual edit before first run

Before running the pipeline, valid HMD credentials must be supplied.

The synthetic data generator accesses Italy life-table data from the Human Mortality
Database (HMD). To reproduce the results, define the environment variables `HMD_USER`
and `HMD_PASS` locally before running the pipeline. These credentials are not included
in the supplement zip.

In the current version of `_targets.R`, the HMD credentials are stored in the targets

```r
tar_target(hmd_user, "..."),
tar_target(hmd_pass, "...")
```

near the top of the file.

### Temporary local edit for reproduction

Open `_targets.R` and replace the two credential strings with your own HMD username and password.

For example:

```r
tar_target(hmd_user, "YOUR_HMD_USERNAME"),
tar_target(hmd_pass, "YOUR_HMD_PASSWORD"),
```

### Preferred safer edit before submission

For the final cleaned supplement, these two lines should ideally be changed to environment-variable based access, e.g.

```r
tar_target(hmd_user, Sys.getenv("HMD_USER")),
tar_target(hmd_pass, Sys.getenv("HMD_PASS")),
```

and then the variables should be defined locally before running R.

## 6. Installation and execution

Open R in the project root, i.e. in the folder that contains `DESCRIPTION` and `_targets.R`. This supplement includes the `_targets/` folder, i.e. the targets cache from a
previous successful run. It is included intentionally so that users can inspect
intermediate objects, perform spot checks, and regenerate selected downstream outputs
without rerunning the entire pipeline from scratch.

### 6.1 Install the package and required tooling

```r
install.packages(c("devtools", "targets", "tarchetypes"))
devtools::install_local(".", upgrade = "never")
library(credMortality)
```

If additional package installation is needed on a fresh machine, install the missing dependencies reported by R.

### 6.2 Clean rerun from scratch

Because the submission folder currently includes a `_targets/` cache, a user who wants a clean rerun should destroy the existing cache first:

```r
library(targets)
tar_destroy(destroy = "all")
```

Then run the full pipeline:

```r
library(targets)
tar_make()
```

This is the main reproduction command for the supplement.

### 6.3 Retrieve outputs after running

For file targets, the saved path can be queried by

```r
targets::tar_read(<target_name>)
```

For example:

```r
targets::tar_read(fig2a_pdf)
targets::tar_read(lc_mse_superpop_pdf)
targets::tar_read(lc_msep_results_file)
```

The corresponding files will also appear directly in `figures/` and `output/`.

## 7. Mapping from manuscript outputs to pipeline targets

### Important note on sub-population numbering

For some simulation-based figures, the manuscript labels the sub-populations in a different order than the internal file numbering used by the code. In particular:

- file names with `group_2` correspond to **Sub-population 1** in the manuscript
- file names with `group_3` correspond to **Sub-population 2**
- file names with `group_1` correspond to **Sub-population 3**

The same convention applies to the MSEP files:
- `g2` = manuscript Sub-population 1
- `g3` = manuscript Sub-population 2
- `g1` = manuscript Sub-population 3

This README follows the **manuscript naming** first and gives the corresponding target/file explicitly.

## 9. Table in the manuscript

The manuscript currently contains one LaTeX table:

- `tab:label_model_predictor_constraints`

This table is typeset manually in the manuscript source. The identification constraints are textual model definitions and are therefore not generated by the R code. However, the average BIC values used in the table can be reconstructed from the rolling-window simulation output file `output/results_age_breakets_smooth.csv`.

### Reconstructing average BIC values

```r
library(data.table)

dt <- fread("output/results_age_breakets_smooth.csv")

bic_dt <- unique(dt[, .(predictor, seed_ix, years_ix, bic)])

bic_summary <- bic_dt[, .(
  average_bic = mean(bic),
  average_bic_div_1000 = mean(bic) / 1000
), by = predictor]

bic_summary[order(average_bic)]
```

This reproduces the empirical BIC summary from the saved simulation output.

## 12. Notes on reproducibility

The submitted supplement intentionally includes the `_targets/` folder. This folder is
the `targets` cache from a previous successful run of the pipeline and is included to
facilitate reproducibility checks for computationally heavier parts of the analysis.

Including the cache allows users to:

- inspect intermediate objects produced by the pipeline,
- perform spot checks of specific steps or outputs without rerunning the entire study,
- regenerate selected downstream figures and summary files more quickly.

Users who wish to rely on the shipped intermediate results may run the pipeline directly
and allow `targets` to reuse the existing cache where appropriate.

Users who prefer a completely fresh rerun from source can first delete the cache and then
rerun the full pipeline:

```r
library(targets)
tar_destroy(destroy = "all")
tar_make()
```

## 13. Minimal reproduction recipe

For a co-author or reviewer who wants the shortest path to regenerate all code-generated outputs:

```r
install.packages(c("devtools", "targets", "tarchetypes"))
devtools::install_local(".", upgrade = "never")

library(targets)

# optional but recommended for a clean rerun
tar_destroy(destroy = "all")

# after replacing HMD credentials in _targets.R
tar_make()
```

After this completes, the reproduced figure files are in `figures/` and the saved result objects are in `output/`.
